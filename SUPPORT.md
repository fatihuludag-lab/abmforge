# Support

ABMForge is an alpha-stage research software project.

Support is currently provided on a best-effort basis through GitHub.

## Where to Ask

Use GitHub issues for:

- installation problems;
- documentation problems;
- unexpected errors;
- feature requests;
- reproducibility or archive questions;
- model zoo suggestions;
- release or packaging problems.

Use discussions, if enabled, for:

- general questions;
- modelling advice;
- research workflow discussion;
- teaching use cases.

## Before Opening an Issue

Please include:

- ABMForge version;
- Python version;
- operating system;
- installation method;
- command that failed;
- full traceback or error message;
- minimal model or scenario file when possible.

Useful commands:

```bash
python --version
python -m pip show abmforge
abmforge info
abmforge --version
```

## Supported Use Cases

ABMForge is currently most suitable for:

- teaching examples;
- small and medium-sized research models;
- reproducible scenario workflows;
- multi-seed experiments;
- structured datasets;
- validated experiment archives.

## Current Limitations

ABMForge is not yet production-grade for:

- untrusted model execution;
- large distributed simulations;
- guaranteed cross-platform bitwise reproducibility;
- full replay of arbitrary custom model state;
- long-term stable public API guarantees.

## Maintainer Availability

ABMForge is maintained as an open research software project. Response times may
vary depending on teaching, research, and release priorities.

Pull requests with tests and clear motivation are welcome.
